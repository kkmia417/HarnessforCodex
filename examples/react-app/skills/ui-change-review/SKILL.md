# UI Change Review

Review UI changes as user workflows across states and viewport sizes.

## Workflow

1. Identify changed components, routes, state transitions, and data dependencies.
2. Check loading, empty, error, disabled, success, and permission states.
3. Verify keyboard access, labels, focus behavior, and color contrast risks.
4. Check responsive layout for text fit and control stability.
5. Summarize missing tests, visual smoke checks, and release risk.

Read `references/ui-review-checks.md` before producing findings.
