# Maintainer Triage

Turn ambiguous community input into a clear maintainer action.

## Workflow

1. Classify the report as bug, feature request, support, documentation, security, or release blocker.
2. Identify missing reproduction details or decision inputs.
3. Propose labels, priority, owner, and next action.
4. Draft a concise maintainer response when useful.
5. Flag release-blocking risk separately from normal backlog work.

Read `references/triage-rubric.md` before producing a triage summary.
